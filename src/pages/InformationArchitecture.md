# 个人知识库MVP信息架构设计

## 页面清单（4个核心页面）

### 1. 首页（知识库概览）
**页面目标**：提供知识库的整体概览，快速访问核心功能
**核心模块**：
- 快速添加知识入口（悬浮按钮）
- 最近添加的知识条目（3-5条）
- 知识统计信息（总卡片数、分类数、标签数）
- 常用分类快捷入口（最多5个）
- 搜索框（全局搜索）

### 2. 知识列表页
**页面目标**：展示所有知识条目，提供筛选和排序功能
**核心模块**：
- 知识条目列表（卡片式展示）
- 搜索和筛选功能（分类、标签、关键词）
- 排序选项（时间、标题、分类）
- 批量操作功能（选择、删除、移动分类）
- 添加新知识按钮

### 3. 知识详情页
**页面目标**：查看和编辑单个知识条目的完整信息
**核心模块**：
- 知识内容完整展示（标题、内容、分类、标签）
- 编辑和删除功能
- 创建时间、更新时间显示
- 相关联知识推荐（基于分类/标签）
- 返回列表按钮

### 4. 添加/编辑知识页
**页面目标**：创建和编辑知识条目
**核心模块**：
- 表单式知识录入（标题、内容、分类、标签）
- 分类选择器（下拉选择）
- 标签输入（支持多选和新建）
- 保存和取消按钮
- 内容预览功能

## 用户完整路径设计

### 路径1：新增知识流程
1. 用户打开首页
2. 点击"快速添加"按钮或"+"按钮
3. 进入添加知识页，填写标题、内容、选择分类和标签
4. 点击保存，返回首页或列表页
5. 系统提示"知识添加成功"

### 路径2：整理归档流程
1. 用户从首页或列表页查看知识条目
2. 点击某个知识条目进入详情页
3. 点击"编辑"按钮进入编辑模式
4. 修改分类和标签，建立知识关联
5. 保存修改，返回详情页或列表页

### 路径3：回顾复习流程
1. 用户从首页查看最近添加的知识
2. 通过分类快捷入口快速筛选特定类型知识
3. 在列表页使用搜索功能查找相关知识
4. 点击知识条目查看详情，进行复习
5. 可标记重要知识或添加复习笔记

## 页面合并优化策略

### 合并方案1：首页 + 列表页
**优化建议**：将首页和列表页合并为一个"知识库"页面
**实现方式**：
- 页面顶部显示统计信息和搜索框
- 中间部分显示知识列表
- 添加"快速筛选"区域替代原来的分类快捷入口
- 节省1个页面，减少用户导航层级

### 合并方案2：详情页 + 编辑页
**优化建议**：详情页内嵌编辑功能
**实现方式**：
- 详情页默认显示查看模式
- 点击"编辑"按钮切换到编辑模式（同页面）
- 使用模态框或页面内切换实现
- 节省1个页面，提升操作流畅度

### 最终推荐方案：3个核心页面
1. **知识库主页**（合并首页+列表页）
2. **知识详情页**（内嵌编辑功能）
3. **添加知识页**（独立页面，保持添加流程清晰）

## MVP功能优先级

### 核心功能（必须实现）
- 知识的增删改查（CRUD）
- 基础分类管理（预设几个常用分类）
- 简单标签系统
- 全文搜索功能
- 本地数据存储

### 简化功能（MVP版本简化）
- 分类管理：预设5个固定分类，不支持自定义
- 标签系统：支持输入，但不支持颜色和统计
- 搜索功能：基础关键词搜索，不支持高级语法
- 统计信息：仅显示总数，不显示图表

### 可延后功能（V1.1版本）
- 知识关联推荐
- 导入导出功能
- 主题切换
- 高级搜索和筛选
- 知识时间线

## 演示流程设计

### 演示脚本（5分钟）
1. **开场**（30秒）：介绍产品价值 - "解决碎片化知识管理问题"
2. **展示首页**（1分钟）：展示知识库概览，快速添加功能
3. **添加知识**（1.5分钟）：演示添加新知识，选择分类和标签
4. **查看列表**（1分钟）：展示知识列表，使用搜索和筛选
5. **编辑整理**（1分钟）：进入详情页，编辑分类和标签
6. **总结**（30秒）：强调产品核心价值和使用场景
